# ADR-001: All-in-one MVP for Signup/Signin and Multi-Tenant User Management

## Status

Accepted

## Date

2026-04-07

## Context

The product needs multi-tenant support with organization signup, user authentication, and role-based user management. The question was whether to deliver this as a single release or split into multiple phases (auth-first or structure-first).

## Decision

Deliver all signup/signin and multi-tenant user management capabilities in a single MVP phase. This includes: organization signup with initial admin creation, email/password signin, user management (admin creates users directly), organization settings, role-based access (admin boolean), and tenant-scoped ticket types.

## Alternatives Considered

### Alternative 1: Auth-first, then Management

- **Description**: Ship signup/signin and tenant-scoped data first, add user management and org settings in a second phase.
- **Pros**: Smaller initial scope, faster first delivery.
- **Cons**: Organizations are single-user until Phase 2, feels incomplete, requires two deployment cycles.
- **Why rejected**: The feature set is cohesive and interdependent — shipping without user management leaves organizations non-functional for team use.

### Alternative 2: Structure-first, Signin Later

- **Description**: Build org creation, user CRUD, and role-based structure first without authentication, then add signin/security in Phase 2.
- **Pros**: Gets data model right first, allows early workflow testing.
- **Cons**: No security in Phase 1, not deployable to real environments, backward auth retrofit is risky.
- **Why rejected**: Delivering without authentication creates security concerns and prevents real-world usage.

## Consequences

### Positive

- Complete tenant isolation delivered in one release
- All user stories addressed immediately
- Consistent UX from day one
- No temporary workarounds or incomplete states

### Negative

- Larger initial development scope
- No phased risk reduction — all features ship together

### Risks

- Scope creep during implementation — mitigated by strict adherence to defined MVP features
- Longer time to first delivery — mitigated by keeping features minimal (no invitations, no SSO, no password recovery)

## Implementation Notes

Email is globally unique across organizations, enabling simple email + password signin without organization selection.
