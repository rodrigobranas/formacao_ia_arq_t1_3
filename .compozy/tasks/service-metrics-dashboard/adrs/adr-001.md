# ADR-001: Dashboard Product Approach — Real-Time Operations Hub

## Status

Accepted

## Date

2026-04-07

## Context

The team needs a service metrics dashboard for operations managers. The primary goal is real-time queue management — giving managers instant visibility into open tickets, unassigned work, and current wait times. The secondary goal is historical trend analysis. Success will be measured by daily active usage, so the dashboard must be compelling enough to become the default "home screen."

Three product approaches were evaluated: a real-time operations hub, an analytics-first layout with live widgets, and a dual-mode dashboard with separate live and analytics views.

## Decision

Adopt the **Real-Time Operations Hub** approach. The dashboard will be built as a live operations center with auto-refreshing KPI cards at the top, a real-time ticket queue sorted by urgency in the center, and historical trend charts accessible via a secondary tab.

## Alternatives Considered

### Alternative 1: Analytics-First with Live Widgets

- **Description**: Historical charts and trends as the main view, with a sidebar widget for live queue stats.
- **Pros**: Richer analytical insights on first load; better for weekly/monthly reviews.
- **Cons**: Real-time management feels secondary; less compelling as a daily "go-to" screen.
- **Why rejected**: Does not prioritize the primary goal of real-time queue management and is less likely to drive daily active usage.

### Alternative 2: Dual-Mode Dashboard

- **Description**: Two fully separate toggleable modes — "Live" (real-time queue, KPIs) and "Analytics" (historical charts, trends).
- **Pros**: Best of both worlds with no compromises on either use case.
- **Cons**: Essentially two dashboards, significantly larger MVP scope; mode switching adds daily friction.
- **Why rejected**: Scope too large for MVP. The added complexity and toggle friction reduce the likelihood of it becoming a seamless daily tool.

## Consequences

### Positive

- Directly addresses the primary need for real-time queue visibility
- Highest likelihood of becoming the daily-use "home screen" for operations managers
- Manageable MVP scope — one primary view with a secondary tab

### Negative

- Historical analytics are secondary, requiring a tab switch
- May need to evolve toward dual-mode in a later phase if analytics demand grows

### Risks

- Auto-refresh mechanism must be performant to avoid degrading the user experience
- If historical analytics prove equally important, the tab-based secondary placement may frustrate users — mitigated by monitoring usage patterns and iterating in Phase 2
