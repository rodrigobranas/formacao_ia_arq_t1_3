# ADR-004: SQL Init Script for Database Schema via Docker Compose Volume

## Status

Accepted

## Date

2026-04-06

## Context

The backend connects to PostgreSQL via pg-promise. We need to decide how the initial database schema (tables, etc.) is created. Students need a simple, understandable mechanism that works reliably on first setup.

## Decision

Include a `database/init.sql` file at the project root containing the initial schema (CREATE TABLE statements). This file is mounted into the PostgreSQL container via Docker Compose volumes, using the `/docker-entrypoint-initdb.d/` directory which PostgreSQL executes on first database initialization.

## Alternatives Considered

### Alternative 1: Programmatic schema setup in application code

- **Description**: Add a setup function in the backend that creates tables on first run using pg-promise queries.
- **Pros**: No external SQL files, everything in TypeScript.
- **Cons**: Mixes schema management with application code, runs on every startup (needs idempotency guards), harder to inspect the schema at a glance.
- **Why rejected**: Violates separation of concerns; students should see schema as a separate, inspectable artifact.

### Alternative 2: Dedicated npm seed script

- **Description**: A `db:setup` npm script that reads SQL files and executes them against the database.
- **Pros**: Explicit manual step, flexible.
- **Cons**: Adds another step to the setup process, requires the backend to be partially configured to run.
- **Why rejected**: Adds friction to initial setup; Docker Compose's init mechanism handles this automatically.

## Consequences

### Positive

- Schema is visible in a single SQL file — students can read and understand it directly
- Automatic execution on `docker compose up` — zero extra steps
- Clean separation between schema definition and application code
- Students learn raw SQL from the start

### Negative

- Schema changes require recreating the Docker volume (`docker compose down -v && docker compose up -d`)
- No migration history or versioning

### Risks

- Students may forget to recreate the volume after schema changes; mitigated by documenting this in the README and adding a `make db:reset` target

## Implementation Notes

Docker Compose will mount `./database/init.sql` to `/docker-entrypoint-initdb.d/init.sql`. The SQL file will contain minimal schema (e.g., a single example table) to keep the skeleton lightweight.
